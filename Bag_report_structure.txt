Bag-reports— документ содержащий отчет о люом недостатке к компоненте или системе, который может привести компонент или сестему к невозможному выполнению требуемой функции. 

Основные поля баг-репорта 
1) Уникальный индентификатор (ID)
2) Описание (Sammary): краткое, емкое и понятное описание (Что? Где? При каких условиях) 
3) Окружение (Environment): ссылка на билд/коммит/версия ПО и всего окружения
4) Шаги воспроизведения (Steps to reproduce): полный перечень шагов для воспроизведения
5) Ожидаемый результат (Expected result): какой результат должен был быть без дефекта
6) Фактический результат (Actual result): какой результат получился на самом деле
7) Вложения (Attachments): логи, скриншоты, видео- все что необходимо для понимания дефекта 

Дополнительные:
8) Предварительные условия (Prerequisites) 
9) Тестовые данные(Test data)
10) Серьерзность дефекта (defect severity)
11) Комментарии (Remarks)
12) Проект 
13) Продукт 
14) Версия релиза 
15) Модуль
16) Обнаружение в версии
17) Вероятность возникновения дефекта
18) Приоритет дефекта 
19) Автор отчета 
20) Назначенл на 
21) Статус 
22) Fixed build version
